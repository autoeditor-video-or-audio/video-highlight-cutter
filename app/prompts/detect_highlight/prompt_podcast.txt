SAÍDA (OBRIGATÓRIA)
- Responda SOMENTE com um ARRAY JSON (comece com [ e termine com ]).
- É PROIBIDO responder com objeto JSON (ex.: {} ou {"cortes":[...]}). Se precisar retornar algo, responda apenas o ARRAY.
- "start" e "end" em segundos decimais (máx. 1 casa), ordenados por "start", sem sobreposição.
- Se não houver cortes válidos: [].

OBJETIVO (PODCAST)
- Produzir cortes ≤ 90.0 s que capturem uma micro-conversa completa, com início → desenvolvimento → fechamento.
- Cada corte deve fazer sentido isoladamente, como se fosse um trecho pronto para Shorts/Reels.
- Nunca dividir por tempo fixo; só por mudança de assunto, piada concluída ou conclusão natural.

REGRAS DURAS
R1. Uma conversa/ideia = um corte.  
R2. Nunca encerrar em frase incompleta. Encerrar após ponto final, interrogação, exclamação OU punchline/reação.  
R3. Adicionar PAD = 0.8 s após fechamento para incluir pausa ou reação dos participantes.  
R4. Máx. 90.0 s por corte. Se uma conversa exceder isso, selecione subtrecho coeso (pergunta→resposta, setup→piada) que caiba no limite.  
R5. É proibido criar cortes de duração regular (ex.: blocos de 30s/45s). Cada corte deve existir por causa do conteúdo.  
R6. Não encerrar se a próxima fala começar em menos de 0.5 s; estenda ou combine no mesmo bloco.

GATILHOS DE PODCAST (MOMENTOS QUE DEVEM VIRAR CORTE)
- Pergunta e resposta completa.  
- Histórias curtas contadas de início ao fim.  
- Piadas, risadas ou improvisos concluídos.  
- Discussões ou debates acalorados que fecham em uma conclusão.  
- Frases de impacto, insights, opiniões fortes ou revelações.  
- Reações emocionais claras (risada, surpresa, indignação, inspiração).

REGRAS PARA .SRT
- Converter “HH:MM:SS,mmm” → segundos (1 casa).  
- Cues consecutivos:  
  • gap < 1.0 s ⇒ mesma fala/ideia.  
  • 1.0 s ≤ gap < 2.0 s e ainda sem fechamento ⇒ manter no mesmo corte.  
  • gap ≥ 1.5 s + nova pergunta/assunto ⇒ iniciar novo corte.  

FECHAMENTO
- Encerrar após conclusão clara, punchline, ou reação finalizada.  
- Sempre incluir a reação dos participantes (risada, concordância, aplauso) quando ocorrer logo após a fala.  
- Adicionar PAD = 0.8 s após fechamento.  
- Nunca encerrar em palavra ou frase pela metade.

VALIDAÇÃO
- Cada corte deve conter uma micro-conversa completa.  
- Nenhum corte pode ser baseado apenas em tempo fixo.  
- ≤ 90.0 s; ordenados; sem sobreposição.  
- Se a saída tiver ≤ 2 cortes e DURATION > 120.0 s, reavaliar segmentação para incluir mais tópicos relevantes.  
- Responder estritamente com um ARRAY JSON. Se nada atender, retorne [].
